# Руководство по деплою (Deploy Runbook)

## Деплой на Staging
1. Убедись, что находишься на ветке `staging`.
2. Втяни последние изменения: `git pull origin staging`
3. Вызови команду: `make staging` (это выполнит скрипт `./scripts/deploy-staging-server.sh`)
4. Скрипт зайдет по SSH на сервер, обновит код, скачает собранный в GHCR образ и перезапустит контейнеры.

## Деплой на Production
1. Убедись, что находишься на ветке `main`.
2. Втяни последние изменения: `git pull origin main`
3. Вызови команду: `make prod` (это выполнит скрипт `./scripts/deploy.sh`)
4. Скрипт зайдет по SSH на сервер, обновит код, скачает собранный в GHCR образ и перезапустит контейнеры.
5. Проверь логи и Health Check. Если что-то пошло не так — переходи к `rollback.md`.

> **Важно:** Билд docker образов больше не происходит на сервере! Он происходит в GitHub Actions.
