# Security Audit

1. **Утечка PII (Personal Identifiable Information)**
   - В логах могут появляться stack traces из-за отсутствия централизованного error handling.
   - Используются `console.log` / `console.error` вместо структурированного логгера, что может привести к логированию содержимого сообщений (например, при падении запроса в `catch (err)`).
   
2. **Токены (JWT)**
   - JWT токены сейчас хранятся в `localStorage` на клиенте, что делает их уязвимыми для XSS (Cross-Site Scripting) атак.
   
3. **CORS**
   - На проде whitelist origins берется из `process.env.FRONTEND_URL` и `process.env.ADMIN_URL`. Если `ADMIN_URL` не задан, админка на порту 8443 может блокироваться политикой CORS.
   
4. **Rate Limiting**
   - Отсутствует rate limiting на уровне приложения (пакета `express-rate-limit` нет). Есть только ручной IP blacklist, что не защищает от брутфорса паролей/инвайтов и спама сообщениями.
   
5. **Шифрование БД**
   - Переменная `DB_ENCRYPTION_KEY` передается в контейнеры, и используется для шифрования сообщений. Проверка `utils/encryption.js` показала наличие warning, если ключ не задан. Если ключ потерян, все старые сообщения нечитаемы.
   
6. **Секреты в Git**
   - Проверка `git log --all --full-history -- '.env*'` показала, что файлы с секретами в историю не коммитились. Файлы `.env` находятся в локальном окружении и серверах в `.gitignore`.
