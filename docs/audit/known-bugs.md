# Known Bugs

## P0 (Критические блокеры)

1. **Рассинхрон API вызовов фронта и бэка (Чаты vs Беседы)**
   - **Шаги воспроизведения:** Открыть интерфейс чата, отправить сообщение, дождаться ответа Grok, посмотреть генерацию заголовка.
   - **Ожидаемое поведение:** Название чата генерируется и сохраняется.
   - **Фактическое поведение:** Фронтенд работает с `/api/chats` (legacy), но затем вызывает `/api/conversations/:id/generate-title` (новый роут), что приводит к 404/500 или записи в неверную таблицу, так как ID чатов из разных таблиц не совпадают.

## P1 (Серьезные баги)

2. **Некорректная обработка истечения JWT**
   - **Шаги воспроизведения:** Дождаться истечения токена, попытаться выполнить действие в чате.
   - **Ожидаемое поведение:** Мягкий редирект на логин, сессия очищается корректно.
   - **Фактическое поведение:** Вызов fetch возвращает статус 401, происходит жесткий редирект `window.location.href = '/auth.html'`, данные на форме сбрасываются. Токен остается в `localStorage`, если не почищен.
   
3. **Admin CORS Block**
   - **Шаги воспроизведения:** Попытка залогиниться в админ-панель на production (`https://anita-psy.online:8443`).
   - **Ожидаемое поведение:** Успешный вход.
   - **Фактическое поведение:** В `server.js` `_allowedOrigins` включает только `FRONTEND_URL`. Админка на другом порту/домене блокируется CORS.

## P2 (Минорные баги)

4. **Заглушки try/catch "table may not exist"**
   - **Ожидаемое поведение:** Миграции создают схему, приложение просто делает запросы.
   - **Фактическое поведение:** Запросы оборачиваются в пустые catch блоки, скрывая возможные реальные ошибки подключения или синтаксиса.
